Добавь отдельную страницу товара по маршруту /product/:id:
Роутинг:

Новый маршрут в React Router: /product/:id, где id — это id товара из базы данных.

API:
2. Новый GET эндпоинт /api/products/:id — возвращает один товар по id (только published = true).
Страница товара — содержимое:
3. Фото товара крупно (карусель если несколько фото, с поддержкой свайпов на мобильных — как в карточках каталога).
4. Бренд, название, размер (или «Длина стельки» для обуви), цена.
5. Ироничная подпись (caption) если есть.
6. Таблица размеров (аккордеон) если категория «Обувь» — как в карточках каталога.
7. Кнопка «Купить на Авито» если есть avitoLink, иначе «Написать в Telegram». Логика проданных/забронированных — как в карточках.
8. Лента «ПРОДАНО» поверх фото если товар продан.
9. Кнопка «← Назад в каталог» сверху.
SEO мета-теги (динамические):
10. Для каждого товара генерировать в <head>:
- <title> — «{Название} {Бренд} — PickMe Store»
- <meta name="description"> — «{Бренд} {Название}, размер {size}. Цена {price} ₽. Оригинал. Доставка по России»
- og:title — то же что title
- og:description — то же что description
- og:image — URL главного фото товара (https://pickmestore.ru/uploads/...)
- og:url — https://pickmestore.ru/product/{id}
- og:type — product
- twitter:card — summary_large_image
- Так как сайт SPA, мета-теги нужно обновлять через JavaScript (document.title, метатеги через DOM) при загрузке страницы товара.
Ссылки на страницу товара:
11. В каталоге (/catalog) и на главной — стрелка (→) в карточке товара теперь ведёт на /product/{id} вместо текущего поведения.
12. Карточка товара в каталоге тоже кликабельная — клик по фото или названию ведёт на /product/{id}.
Дизайн:
13. Розовая стилистика сайта, glassmorphism-карточка на фоне общего градиента. На десктопе — фото слева (50%), информация справа (50%). На мобильных — фото сверху, информация снизу.
14. Страница должна работать при прямом переходе по URL (для соцсетей и поисковиков).
Важно:
15. Обнови Яндекс.Метрику — при переходе на /product/:id отправлять hit.
16. Обнови аналитику — при открытии страницы товара записывать card_view.
