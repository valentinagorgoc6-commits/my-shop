# PickMe Store — Решения и их причины

Этот файл фиксирует ключевые архитектурные и дизайнерские решения, чтобы в будущем было понятно, почему выбран тот или иной подход.

---

## Стек

### Почему React + Express, а не Next.js?
Проект начинался в Replit Agent, который сгенерировал эту связку. Работает стабильно, причин менять нет. SSR не критичен — SEO закрывается через OG-теги и sitemap.

### Почему Drizzle ORM, а не Prisma?
Выбор Replit Agent при генерации. Drizzle легковесный и хорошо работает с PostgreSQL. Миграции через `drizzle-kit push`.

### Почему pnpm, а не npm/yarn?
Монорепо с workspaces — pnpm лучше всего подходит для этого. Быстрее, экономнее по диску.

### Почему PM2?
Простой процесс-менеджер для Node.js. Автоперезапуск при падении, логирование. Для одного сервера — оптимальный выбор.

## Дизайн

### Почему три темы, а не две?
Мужская тёмная тема добавлена для комфорта просмотра. Переключатель свет/тьма — стандартный UX, который ожидают пользователи. Женская тема только светлая, потому что розовая стилистика плохо ложится на тёмный фон.

### Почему splash screen, а не автоопределение?
Невозможно определить пол посетителя автоматически. Splash screen — явный выбор пользователя. Не показывается на /product/ и /gift/ чтобы не мешать прямым переходам по ссылкам.

### Почему Caveat только в женской теме?
Caveat — рукописный, «девчачий» шрифт. В мужской теме выглядит неуместно. Заменён на Montserrat через CSS-переменную --font-accent.

### Почему страница /gift/:id отдельная, а не параметр?
Подарочная ссылка — это отдельный UX: без цены, без кнопок покупки, всегда в женской теме. Отдельный роут проще контролировать, чем условную логику на /product/.

## Инфраструктура

### Почему VPS, а не Vercel/Railway?
Российский VPS (reg.ru) — быстрее для российской аудитории. Полный контроль над сервером. Дешевле при долгосрочном использовании.

### Почему Nginx перед Express?
Стандартная практика: Nginx раздаёт статику (фото, CSS, JS) эффективнее Node.js. Express обрабатывает только API и SSR-роуты.

### Почему фото в /uploads/, а не в S3?
Простота. Для текущего объёма товаров (~100 позиций) хранение на диске достаточно. Бэкапы через scp. Переход на S3 имеет смысл при масштабировании.

### Почему перешли с Replit на Claude Code?
Replit Agent имеет ограничения по времени работы и контролю. Claude Code даёт полный контроль, работает локально, изменения сразу видны в git. Удобнее для итеративной разработки.
